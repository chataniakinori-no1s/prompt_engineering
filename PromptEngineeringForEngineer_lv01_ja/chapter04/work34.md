# ブラウジング技術選定—要件整理から比較表作成まで
:::sample-quiz

## サンプル課題
田中さんとの打ち合わせを終えて自社に戻ろうとすると、デスクに座っていた社員と目が合い「もしかしてAIコンサルタントの方ですか！？」と呼び止められました。 あなたに声をかけたのは、今年GrowthTechに入社して4年目になるシステムエンジニアの高橋さんです。 営業担当から次のような相談を受けて、頭を抱えているようです。

>高橋さんのお悩み: 
>「営業担当から『顧客がリアルタイム翻訳機能を欲しがっているんだけど、うちのシステムに組み込める？』という相談が来たんです。正直、翻訳APIって何があるのかもよく分からないし、どれを選べばいいのか見当もつきません。料金やセキュリティも気になるし、最新の情報を調べる時間もなくて…」

このような技術選定の段階でも、AIのウェブ検索機能が威力を発揮します。ChatGPTやClaudeの無料アカウントでも使えるブラウジング機能を活用して、高橋さんの技術調査を加速させましょう。 
:::

## 解説
### ブラウジング対応AIで最新情報をリサーチ
ChatGPTやClaudeなどのAIモデルは、無料アカウントでもウェブ検索機能を使うことができます。この機能を活用すると、以下のような調査を効率的に進められます。

候補の洗い出し: 「リアルタイム翻訳API」という漠然とした要件から、実在するサービスをリストアップ
最新情報の取得: 料金改定やサービス終了など、日々変化する情報を公式サイトから収集
比較表の自動生成: 複数のサービスを指定した観点で比較し、意思決定に必要な情報を構造化
根拠URLの提示: 調査結果の信頼性を担保するため、情報源となった公式ページのリンクを明記

### ウェブ検索機能の使い方（ChatGPTの場合）
1. `+`をクリック
2. `さらに表示`→`ウェブ検索`を選択

![](https://chataniakinori-no1s.github.io/prompt_engineering/PromptEngineeringForEngineer_lv01_ja/assets/chapter04/work34-1.png "")

### ウェブ検索機能の使い方（Claudeの場合）
1. `検索とツール`ボタンをクリック
2. `ウェブ検索`をオンにする

![](https://chataniakinori-no1s.github.io/prompt_engineering/PromptEngineeringForEngineer_lv01_ja/assets/chapter04/work34-2.png "")

:::Point

### ブラウジング結果も必ずクロスチェック
AIのウェブ検索機能は便利ですが、以下の点に注意が必要です：

情報の鮮度: AIが参照したページが最新でない可能性があります
解釈の誤り: AIが公式情報を誤って理解している場合があります
複数ソースでの確認: 重要な情報（料金、SLA、セキュリティ）は必ず複数の情報源で確認しましょう
特に料金やライセンスに関わる情報は、提示されたURLを実際に開いて、自分の目で確認することを推奨します。 
:::

:::Warning
AIが提示する情報は鵜吞みにすることなく、調査の「とっかかり」として活用してください。正式な技術選定や契約に関わる判断は、必ず公式ドキュメントや営業担当者への問い合わせを通じて行ってください。 
:::

## 4. 推奨案 & 理由

このプロジェクト（日本語／英語双方向、月 100 万文字、リアルタイム性重視）に対して、最適だと考えるサービスを一つ挙げるとすれば、**DeepL API（Pro）** を推奨することが多いと思います。ただし、以下の点は貴社の状況（クラウド基盤、コスト許容、リージョン要件など）に依存します。

### 推奨：DeepL API（Pro）

#### 推奨理由（3点）

1. **日本語 ↔ 英語間での高品質な翻訳**
   　DeepL は特に日本語・英語間において、読みやすさや自然な言い回しに強みがあると、多くのユーザーレビューで評価されています。([smartling.com][10])
   　リアルタイム翻訳で「ぎこちない訳」ではユーザー体験を損ないやすいため、この品質の優位性は重要です。

2. **明示的なデータ非保存・プライバシーポリシー**
   　DeepL Pro（API 側）では「翻訳後に即削除」「入力テキストを学習に利用しない」ポリシーが明記されており、データプライバシー重視の要件を満たしやすい。([DeepL Help Center][11])
   　多くの企業で「内部文書やユーザーデータを送る」ような翻訳用途では、この点が選定判断において重視されます。

3. **無制限の文字数対応とコスト制御の柔軟性**
   　DeepL API Pro は定額課金＋従量課金方式で、文字数制限がないことをうたっており（つまり、ボトルネックなく拡張可能）([DeepL Help Center][11])
   　月 100 万文字規模であれば現実的なコストで収まりやすく、将来的な拡張にも耐えられる設計と見られます。

#### 注意・補足条件（判断要素）

ただし、DeepL を選ぶ際には以下点もチェックすべきです：

* リージョン対応：DeepL が日本国内／アジア地域に十分なレスポンス性能を出せるか
* 同時翻訳リクエスト数とスループット性能（API の制限）
* コスト比較：100 万文字レベルで他サービス（特に Azure Translator や Google 翻訳 API）と比較した時点での単価差
* 契約条件（SLA、サポート、可用性保証、障害対応）
* 用語カスタマイズ性（辞書登録・用語制御機能）が必要な場合、その対応可否

もし、御社がすでに AWS や Azure を採用しており、統合運用性を重視するなら、**Amazon Translate** や **Microsoft Translator** のほうが選択肢として非常に強いです。

（たとえば、AWS 環境下であれば Amazon Translate は他サービスとの親和性が高く、セキュリティ設定も統合しやすいという利点があります。([AWS ドキュメント][7])）
